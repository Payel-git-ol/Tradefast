**Промт для создания краулера на Playwright:**

```
Напиши полный рабочий код краулера на TS с использованием Playwright, который выполняет следующий алгоритм:

1. **Начальная точка:** Краулер начинает с указанной стартовой URL-адреса (например, https://ru.investing.com/economic-calendar)

2. **Краулинг ссылок:** 
   - Открывает страницу с помощью Playwright
   - Сканирует весь HTML-контент страницы на наличие всех ссылок (<a href="...">)
   - Извлекает все валидные URL-адреса (относительные и абсолютные)
   - Нормализует относительные ссылки в абсолютные (добавляет домен)
   - Фильтрует ссылки, оставляя только те, которые относятся к тому же домену (ru.investing.com)

3. **Рекурсивный обход:**
   - Для каждой найденной ссылки, которой ещё не было обработано:
     - Переходит по этой ссылке
     - Повторяет процесс сканирования HTML на наличие новых ссылок
     - Добавляет новую страницу в очередь для обработки
   - Продолжает до тех пор, пока не достигнет "тупика" (нет новых ссылок для перехода)

4. **Требования к реализации:**
   - Использовать Playwright для рендеринга JavaScript-страниц
   - Реализовать очередь для обработки страниц (BFS - breadth-first search)
   - Хранить посещённые URL в Set для избежания циклов и повторных посещений
   - Добавить задержку между запросами (2-3 секунды) для избежания блокировки
   - Обработать ошибки сети и таймауты
   - Ограничить максимальное количество страниц для краулинга (например, 100 страниц)
   - Сохранять все найденные ссылки в массив/файл
   - Добавить логирование процесса (какие страницы посещены, какие ссылки найдены)

5. **Дополнительные функции:**
   - Извлекать только ссылки внутри основного контента (игнорировать навигацию, футер, если возможно)
   - Фильтровать ссылки по типам (игнорировать mailto:, javascript:, внешние домены)
   - Сохранять структуру пути краулинга (какая ссылка привела к какой странице)
   - Добавить возможность приостановки и продолжения краулинга

6. **Пример ожидаемого поведения:**
   ```
   https://ru.investing.com/economic-calendar
     ↓
   https://ru.investing.com/economic-calendar/brc-shop-price-index-19
     ↓
   https://ru.investing.com/analysis/article-200297782
     ↓
   (нет новых ссылок → тупик, переход к следующей ссылке из очереди)
   ```

7. **Технические требования:**
   - Язык: JavaScript (Node.js)
   - Библиотека: Playwright (из коробки)
   - Формат вывода: JSON файл со всеми найденными ссылками и их путями
   - Обработка ошибок: try-catch для каждого запроса
   - Конфигурация: максимальное количество страниц, задержка, таймаут - как переменные

Напиши полный код с комментариями на русском языке, готовый к запуску после установки зависимостей (npm install playwright).
```
